spring_ioc

Posted on 2020-11-18 Edited on 2020-11-23

IOC（控制反转）：解决类之间的依赖
将创建对象的权利交给工厂或者框架 主要是解决程序间的耦合（降低程序间的依赖）
原始做法：创建一个beanFactory来负责创建和注入对象
-需要一个配置文件配置所需要用到的类
配置内容: 唯一标识 = 全限定类名 类似Map里的（key-value）形式
-通过读取配置文件的内容，反射创建对象
-使用HashMap来存储各个对象（key=唯一标识，value = 相应的object）

Spring是一个框架可以帮我们做上述的事情
一般地我们需要用ApplicationContext来获取核心容器对象
ApplicationContext的三个实现类：
ClassPathXmlApplicationContext：可加载类路径下的配置文件
FileSystemXmlApplicationContext：可加载在任意磁盘下路径的配置文件
AnnotationConfigApplicationContext：用于读取注解来创建容器的
ApplicationContext vs BeanFactory
ApplicatinContext在创建核心容器时，采用的是立即创建模式，一读取完配置文件立马创建 立即创建策略（单例对象适用）
BeanFactory在创建核心容器时采用的是延迟加载模式，什么时候根据id获取对象了，什么时候才创建 延迟加载策略（多例对象适用）
创建bean的三种方式
第一种方式：使用默认构造函数
在spring的配置文件中配以id和class属性之后，且没有其他属性和标签时
采用的是默认构造函数创建bean，如果没有默认构造函数，则无法创建
第二种方式：使用普通过场模式的方法创建对象

第三种方式：使用工厂的静态方法创建对象（使用某各类中的静态方法创建对象，并存入spring容器)

  • 还有一道理论题就是 线程与进程的优点，缺点以及应用场景

    进程因为要单独分配一个独立的地址空间，系统开销会比较大， 因为多进程相互独立
    所以进程间通信比较麻烦 需要通过IPC

    线程是共享数据段的，多线程间使用相同的地址空间，共享大部分数据
    所以在通信方面要比进程方便得多 并且上下文切换也比较容易 但是相比较于进程不够安全
    如果一个进程崩溃 其他进程不会收到干扰 但一个线程崩溃 可能会导致整个进程崩溃

    进程与线程的选择取决以下几点：1、需要频繁创建销毁的优先使用线程；因为对进程来说创建和销毁一个进程代价是很大的。2、线程的切换速度快，所以在需要大量计算，切换频繁时用线程，还有耗时的操作使用线程可提高应用程序的响应3、因为对CPU系统的效率使用上线程更占优，所以可能要发展到多机分布的用进程，多核分布用线程；4、并行操作时使用线程，如C/S架构的服务器端并发线程响应用户的请求；5、需要更稳定安全时，适合选择进程；需要速度时，选择线程更好。

Tomcat&Servelet

Posted on 2020-11-09 Edited on 2020-11-18

Tomcat: Apache基金组织 比较常用的服务器 中小型 开源免费 仅支持少量的JavaEE规范（共13种JavaEE规范） JSP/Servelet

* Tomcat部署：
    1. 需要把项目文件夹放进webapps路径下
    2. 将项目页面打包成war放到webapps路径下
    3. 如果在别的目录下，可以在conf/server.xml 用<Context docBase= "项目存放路径" path= "虚拟目录"/>进行配置
    4. 为降低各个项目之间耦合可在conf/Catalina/localhost路径下创建一个配置文件 虚拟目录=该配置文件的名称
* Tomcat启动：打开terminal 运行startup.sh
* Tomcat关闭：运行shutdown.sh/control + c



* 静态项目和动态项目
    * Java动态项目的目录结构
        --项目的根目录
            --WEB-INF目录
                -- web.xml: web项目的核心配置文件
                -- classes目录：放置字节码文件的目录
                -- lib： 放置依赖的jar包




* Servlet（运行在服务器端的小程序）
    * Servlet就是一个接口，定义了java类被服务器（Tomcaat）识别规范
    * 需要自己重写Servlet接口，实现里面的方法
    * 执行原理
        当服务器接收到客户端请求，解析url获取访问Servlet的资源路径
        查找web.xml 是否有对应的url标签题内容
        如果有找servlet-class全类名
        tomcat加载字节码进内存，并创建对象
        调用service方法
    由于servlet生命周期是项目开始直至项目结束，在此期间servlet可能有多个请求，因此
    servlet不是线程安全的，所以需要注意尽量避免有servlet类成员变量

JavaWeb基础知识笔记

Posted on 2020-11-09 Edited on 2020-11-27

JavaWeb学习笔记

* 软件架构 :
    1. C/S: Client/Server
        * 优点：
            1. 客户端可以做更多服务，用户体验更好
        * 缺点： 
            1. 安装开发部署以及后期维护都很麻烦
    2. B/S: Browser/Server

        * 优点：
            1. 相对于B/S 安装开发部署维护更简洁方便
        * 缺点：
            1. 如果应用过大，用户体验会下降
            2. 对硬件要求过高（服务器的容量以及用户的带宽）
* 服务器端资源分类：
    1. 静态资源： 使用静态开发技术发布的资源
        * 特点：
            * 所有用户访问得到的结果都是一样的
            * 如文本，视频，音频， 图片，HTML， CSS， JAVASCRIPT(动态效果)
            * 无法与数据库交互

    2. 动态资源： 使用动态网页开发技术时发布的资源
        * 特点：
            * 所有用户访问得到的结果可能是不一样的
            * JSP/Servelet，php， asp， ...
            * 如果用户请求的是动态资源， 服务器会先转换为静态资源在发送给浏览器
* 各个静态资源职责
    * HTML ： 用于搭建基础网页，展示网页内容（超文本标记语言）
    * CSS ： 用于美化页面，布局 
    * JavaScript ： 控制页面元素，让其有一些动态效果
* HTTP协议
    * HTTP请求
    

    * HTTP响应
